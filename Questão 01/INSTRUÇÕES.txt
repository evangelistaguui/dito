RACIOCINIO DA QUESTÃO 01

PARA DESCOBRIR AS CINCO MAIORES PESSOAS QUE GERAM MAIS RECEITAS NO BANCO DISPONIBILIZADO,
CRIEI UMA CONSULTA PARA LISTAR OS 5 MAIORES ID COM SUAS RESPECTIVAS RECEITAS, POIS OS EVENTOS DO TIPO TRACK É ARMAZENADO O ID DE QUEM EFETUOU A COMPRA, SENDO QUE ELE NÃO RETORNA NEM NOME, E-MAIL E TELEFONE.
APOS LISTAR OS IDS, CRIEI UM NOVO SELECT UTILIZANDO A FUNÇÃO MAX PARA BUSCAR OS ULTIMOS REGISTRO ATUALIZADOS DE NOME, E-MAIL E TELEFONE UMA VEZ QUE TEM VARIOS REGISTROS DO TIPO IDENTIFY QUE REFERE A ALTERAÇÕES DAS INFORMAÇÕES SOLICITADAS, SENDO ASSIM ELE ESTA RETORNANDO O REGISTRO DE ACORDO COM A ULTIMA ATUALIZAÇÃO.


RACIOCIONIO DA QUESTÃO 02
PRIEMEIRAMENTE EXECUTIE UM SELECT PARA PEGAR A PRIMEIRA DATA DA COMPRA E A ULTIMA DATA DA COMPRA PARA VER A JANELA DE QUANTO TEMPO TINHAS OS DADOS DISPONIBILIZADOS, OU SEJA, A JANELA DISPONIBILIZADA FOI DE UM MÊS.
DEPOIS DISSO EXECUTEI A CONSULTA COM A FORMULA DO AVG E EXTRAC PARA TRABALHAR COM OS DIAS COM A CONDIÇÃO DE TYPE = 'TRACK' NA QUAL OBTIVE O VALOR DE 15,86, POREM PARA MINHA SURPRESA QUANDO EXECUTEI A MESMA CONSULTA POREM PARA O TYPE = 'IDENTIFY' APRESENTOU O MESMO VALOR TAMBEM 15,86.
ENTÃO PARA TER CERTEZA DE QUE A CLAUSULA WHERE ESTAVA SENDO RESPEITADA CRIEI UMA CONSULTA LIMITANDO A CINCO VALORES, SOMEI VALORES NA MÃO E CALCULEI A MEDIA E TIVE O MESMO RESULTADO.
OBS: DEIXEI TODOS OS PASSOS EXECUTADOS NO SQL DA QUESTÃO DOIS SEPARADOS POR ARQUIVOS INDIVIDUAIS.
